# LegalEagle Backend — Setup & Usage

A modular FastAPI service for AI-powered Indian legal research, extracted from the original Streamlit monolith.

## Architecture

```
backend/
  app/
    main.py              ← FastAPI app, lifespan, CORS
    routes/
      chat.py            ← POST /chat
      upload.py          ← POST /upload
      health.py          ← GET /health
    services/
      rag/
        embedder.py      ← InLegalBERT embeddings
        retriever.py     ← FAISS vector store
        loader.py        ← Document ingestion (PDF/DOCX/TXT)
        pipeline.py      ← RAG orchestrator
      llm/
        generator.py     ← Mistral-7B via CTransformers
    models/
      schemas.py         ← Pydantic request/response models
    core/
      config.py          ← Settings + model validation
```

## Prerequisites

- Python 3.10+
- ~8 GB RAM (for Mistral-7B + InLegalBERT)
- ~6 GB disk space for model weights

## Setup

### 1. Install dependencies

```bash
cd LegalEagle
pip install -r backend/requirements.txt
```

### 2. Download model files

The backend requires two local model files that are not included in the repository:

#### InLegalBERT (embedding model)

```bash
huggingface-cli download law-ai/InLegalBERT --local-dir models/InLegalBERT
```

This should produce:
```
models/InLegalBERT/
  config.json
  pytorch_model.bin          ← Required (~440 MB)
  tokenizer_config.json
  special_tokens_map.json
  vocab.txt
```

#### Mistral-7B GGUF (language model)

```bash
huggingface-cli download TheBloke/Mistral-7B-Instruct-v0.2-GGUF \
  mistral-7b-instruct-v0.2.Q4_K_M.gguf \
  --local-dir models/Mistral-7B-Instruct-v0.2-GGUF
```

This should produce:
```
models/Mistral-7B-Instruct-v0.2-GGUF/
  mistral-7b-instruct-v0.2.Q4_K_M.gguf    ← Required (~4.4 GB)
  config.json
```

### 3. Verify FAISS index

The pre-built FAISS index should already exist:
```
LegalKB_FAISS/ipc_embed_db/
  index.faiss
  index.pkl
```

If missing, the backend will rebuild it from `mini_dataset/` on first startup.

## Running

### Production mode (with models)

```bash
cd LegalEagle
python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000
```

### Demo mode (without models)

If model files are not available, run in demo mode which uses mock responses:

```bash
set LEGALEAGLE_DEMO_MODE=true
python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000
```

### Development mode (with auto-reload)

```bash
set LEGALEAGLE_DEBUG=true
python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload
```

## API Endpoints

### `GET /health`
Returns backend status, model load state, and any missing model files.

```bash
curl http://127.0.0.1:8000/health
```

### `POST /chat`
Send a legal question, receive a grounded answer with sources.

```bash
curl -X POST http://127.0.0.1:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "What is Section 420 IPC?"}'
```

Response:
```json
{
  "answer": "Section 420 IPC deals with cheating and dishonestly...",
  "sources": [
    {
      "id": "src-abc123",
      "title": "Indian Penal Code Reference",
      "excerpt": "Whoever cheats and thereby dishonestly induces...",
      "type": "statute"
    }
  ],
  "confidence": "high"
}
```

### `POST /upload`
Upload a PDF/DOCX/TXT file for indexing.

```bash
curl -X POST http://127.0.0.1:8000/upload \
  -F "file=@document.pdf"
```

### `GET /docs`
Interactive Swagger API documentation (auto-generated by FastAPI).

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `LEGALEAGLE_HOST` | `0.0.0.0` | Server bind address |
| `LEGALEAGLE_PORT` | `8000` | Server port |
| `LEGALEAGLE_DEBUG` | `false` | Enable debug mode + auto-reload |
| `LEGALEAGLE_DEMO_MODE` | `false` | Use mock models (no weights needed) |
| `LEGALEAGLE_CORS_ORIGINS` | `["*"]` | Allowed CORS origins |
| `LEGALEAGLE_LLM_TEMPERATURE` | `0.7` | LLM generation temperature |
| `LEGALEAGLE_LLM_MAX_NEW_TOKENS` | `512` | Max generated tokens |
| `LEGALEAGLE_RETRIEVAL_TOP_K` | `3` | Number of documents to retrieve |

## Frontend Connection

The Next.js frontend connects to the backend via:

```env
# frontend/.env.local
NEXT_PUBLIC_API_URL=http://127.0.0.1:8000
```

If not set, the frontend defaults to `http://127.0.0.1:8000`.

## Troubleshooting

### "MODEL FILES MISSING" on startup
Run `/health` to see which files are missing, then download them using the commands above.

### Backend starts but chat returns 503
The RAG pipeline hasn't initialized. Check logs for model loading errors.

### FAISS index not found
The backend will attempt to rebuild from `mini_dataset/`. Ensure the dataset files exist.
